The Hybrid Pattern

Local pattern-matching, cloud judgment

üè†

Stage 1: Local heavy lifting

Use an 8B‚Äì14B coder model in Ollama for scanning repos, scoring snippets, and extracting structure. These are deterministic, pattern-heavy tasks that don‚Äôt need frontier creativity.

‚òÅÔ∏è

Stage 2: Cloud synthesis

Send the distilled context to Claude Code for the judgment call: architecture choices, refactors, final patch. Tiny prompt, big reasoning ‚Äî and a fraction of the tokens.

üîÄ

Router logic

Route based on task type: extraction ‚Üí local; generation ‚Üí cloud. Cache local embeddings; only revalidate deltas. If the cloud call fails, fall back to a smaller local pass to keep moving.

üìâ

Cost profile

Local GPU run: $0 variable, power aside. Cloud synthesis: cents. Combined, you keep latency predictable and make retries free for 80% of steps.

# 4) If cloud fails, retry locally with smaller scope

Setup in 15 Minutes

Keep it sharp, keep it simple

Step 01

Install Ollama + a coder model

Install Ollama, then pull a lightweight coder model (e.g., qwen2.5-coder:8b or llama3.1:8b-instruct). Run it once to warm the cache.

Step 02

Wire Claude Code to the router

Add a simple router: local for scoring/extraction, cloud for synthesis. Keep a max-token ceiling on the cloud leg so retries stay cheap.

Step 03

Cache aggressively

Cache local embeddings and file scores. Re-score only changed files. The less you ship to the cloud, the more predictable your spend.

Step 04

Log everything

Log routed decisions, token counts, and fallbacks. When latency spikes, you should know if the cloud call or the local model is the bottleneck.

Trust and Safety

Local ‚â† automatically safe

üß™

Review the generated code

Local execution removes API exhaust but not logic errors. Keep human review and automated tests in the loop before shipping patches.

üîë

Secrets stay secret

Never paste .env contents into prompts. Mount read-only secrets, redact logs, and keep the router from forwarding sensitive buffers to the cloud leg.

üß≠

Document routing rules

Put the routing policy in your CLAUDE.md: which tasks are local, which are cloud, and token ceilings for each. Agents follow what you write down.

üìä

Watch the SLOs

Track p95 latency and dollar-per-task. Hybrid only wins if it‚Äôs both faster and cheaper. If not, rebalance the split.
