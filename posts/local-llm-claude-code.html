Offline-First AI Local LLMs + Claude Code without hitting rate limits

When Claude hits rate limits, running local models in Ollama keeps you productive. Use local models for repetitive tasks and Claude for the hard problems. This hybrid approach reduces costs and removes the waiting.

The Hybrid Pattern

Local pattern-matching, cloud judgment

üè†

Stage 1: Local processing

Use a small local model for tasks like scanning files, finding patterns, or extracting information. These tasks do not require advanced reasoning.

‚òÅÔ∏è

Stage 2: Cloud reasoning

Send the processed results to Claude for complex decisions like architecture changes or refactoring. The local model reduces the input size and saves tokens.

üîÄ

Simple routing

Decide which tasks go local and which need Claude. Pattern matching and data extraction work fine locally. Design decisions and complex refactors need the cloud model.

üìâ

Lower costs

Local models cost nothing to run after the initial setup. You only pay for Claude when you actually need its capabilities.

Minimal hybrid loop (pseudocode)
# 1) Local scan + score
local = ollama.chat(model="qwen2.5-coder:8b", messages=[
  {"role": "user", "content": "Score these files 1-10 for relevance to `bug: auth timeout`:\n" + file_list}
])

# 2) Compress for cloud
context = summarize(local, budget=1500)  # keep it short

# 3) Send to Claude
claude = anthropic.messages.create(
  model="claude-3.7-sonnet",
  max_tokens=600,
  messages=[{"role": "user", "content": f\"Fix the bug with this context:\\n{context}\"}]
)

# 4) If cloud fails, retry locally with smaller scope
if claude.timed_out:
  fallback = ollama.chat(model="qwen2.5-coder:8b", messages=[...])

Setup Guide

Getting started with local models

Step 01

Install Ollama and a model

Install Ollama, then download a coding model like qwen2.5-coder:8b or llama3.1:8b-instruct. Test it with a simple prompt to make sure it works.

Step 02

Create a simple router

Write a function that decides which model to use. Send simple tasks to the local model and complex reasoning to Claude. Set a token limit for Claude calls to control costs.

Step 03

Cache results

Save results from the local model so you do not process the same files repeatedly. Only reprocess when files change.

Step 04

Track what happens

Log which model handled each task and how many tokens were used. This helps you optimize the routing logic over time.

Quick start example
# You need at least 8GB RAM. A GPU helps but is not required for small models.

# 1) Install Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 2) Pull a small coder model
ollama pull qwen2.5-coder:8b

# 3) Smoke test locally
ollama run qwen2.5-coder:8b "Write a Python function that reverses a string."

# 4) Set your Claude API key
export ANTHROPIC_API_KEY=sk-...your-key...

# 5) Minimal router (Python, no deps)
python - <<'PY'
import requests, os, json
files = "auth.js\nuser.js"
local = requests.post("http://localhost:11434/api/chat", json={
  "model": "qwen2.5-coder:8b",
  "messages": [{"role": "user", "content": f"Score these files 1-10 for auth timeout:\n{files}"}],
})
context = local.json().get("message", {}).get("content", "")[:1500]
cloud = requests.post(
  "https://api.anthropic.com/v1/messages",
  headers={
    "x-api-key": os.environ["ANTHROPIC_API_KEY"],
    "anthropic-version": "2023-06-01"
  },
  json={
    "model": "claude-3.7-sonnet",
    "max_tokens": 600,
    "messages": [{"role": "user", "content": f"Fix auth timeout using:\n{context}"}],
  },
)
print(cloud.json()["content"][0]["text"])

Things to Watch Out For

Local models still need oversight

üß™

Review generated code

Local models can make mistakes just like cloud models. Always review the code and run tests before using it in production.

üîë

Do not share secrets

Never include API keys or passwords in prompts. Make sure your router does not accidentally send sensitive data to external APIs.

üß≠

Document your routing

Write down which tasks use local models and which use Claude. This makes it easier to debug when something goes wrong.

üìä

Measure performance

Track how long tasks take and how much they cost. The hybrid approach should be both faster and cheaper than using only Claude.
