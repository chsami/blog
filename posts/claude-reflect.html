The Problem

Why Manual Memory Management Fails

‚Üª

Repetitive Corrections

Every session, you tell Claude the same things: "use gpt-5.1 not gpt-5", "we use Poetry here", "always add type hints". These corrections vanish when the session ends.

üìù

Manual CLAUDE.md Updates

Context Lost Across Projects

Global preferences (like model versions) need to be in your global CLAUDE.md. Project-specific conventions need to be local. Keeping them organized is tedious.

How It Works

Two-Stage Learning Pipeline

The Reflect Pipeline
1
You Correct Claude
"No, use database for caching"
"Actually, we prefer Ruff"
2
Hook Captures
Automatic detection queues the learning with confidence score
3
/reflect Syncs
Review and approve learnings to CLAUDE.md files
‚ö°

Stage 1: Capture (Automatic)

Hooks run on every prompt, detecting correction patterns like "no, use X", "don't use Y", "actually...", and "that's wrong". Positive feedback like "Perfect!" is captured too. Explicit remember: markers get highest confidence.

‚úì

Stage 2: Process (Manual)

Run /reflect to review queued learnings. An AI-powered semantic filter validates each one, removing false positives. You choose which to apply ‚Äî to global CLAUDE.md, project CLAUDE.md, or AGENTS.md.

03

Detection Methods

Hybrid Pattern Recognition

/.*/

Regex Patterns (Real-time)

Fast pattern matching during sessions detects corrections ("no, use X", "don't use Y"), positive feedback ("Perfect!", "Exactly right"), and explicit markers (remember:). Each pattern has a confidence score from 0.60-0.95.

üß†

Semantic AI Validation

When you run /reflect, an AI-powered semantic filter validates each learning. Multi-language support understands corrections in any language. Extracts concise, actionable statements from natural language.

Multi-Language Support
Corrections in Any Language

The semantic filter understands corrections beyond English patterns. A Spanish correction like "no, usa Python" is correctly detected even though it doesn't match English regex patterns. The final confidence score is the higher of regex and semantic analysis.

Commands Reference

Full Command Suite

Command Description
/reflect Process queued learnings with human review
/reflect --scan-history Scan ALL past sessions for missed learnings
/reflect --dry-run Preview changes without applying them
/reflect --targets Show detected config files (CLAUDE.md, AGENTS.md)
/reflect --review Show queue with confidence scores and decay status
/reflect --dedupe Find and consolidate similar entries in CLAUDE.md
/skip-reflect Discard all queued learnings
/view-queue View pending learnings without processing

06

Smart Features

Beyond Basic Capture

Feature 01

Historical Scan

First time using claude-reflect? Run /reflect --scan-history to scan all past sessions for corrections. Recover learnings from before installation.

Feature 02

Smart Filtering

Questions, one-time task instructions, and vague feedback are automatically filtered out. Only reusable, actionable learnings are kept.

Feature 03

Duplicate Detection

Before adding a learning, existing CLAUDE.md content is checked. If similar content exists, you can merge, replace, or skip.

Feature 04

Semantic Deduplication

Over time, CLAUDE.md accumulates similar entries. Run /reflect --dedupe to consolidate "Use gpt-5.1 for complex tasks" and "Prefer gpt-5.1 for reasoning" into one entry.
