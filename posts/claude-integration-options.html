Claude Integration CLI, API, SDK, and Azure Foundry

As AI becomes central to software development, choosing the right integration method for Claude can significantly impact your project's success. This guide breaks down each option for .NET developers in the public procurement sector.

Production applications needing programmatic AI integration, automated pipelines requiring API responses, or applications needing fine-grained control over parameters.

Claude API

Direct HTTP Access

{}

What is it?

The Claude API is the foundational RESTful interface at https://api.anthropic.com. It's the raw HTTP endpoint that all other integrations ultimately use.

üåê

Universal Compatibility

Works with any programming language that can make HTTP requests. No SDK dependencies required.

üéõÔ∏è

Full Control

Direct access to all parameters, headers, and response handling. Complete flexibility over every aspect of the request.

‚óã

Stateless

Each request is independent; you manage conversation history yourself. Full control over context management.

Basic Example (cURL)
curl https://api.anthropic.com/v1/messages \
  --header "x-api-key: $ANTHROPIC_API_KEY" \
  --header "anthropic-version: 2023-06-01" \
  --header "content-type: application/json" \
  --data '{
    "model": "claude-sonnet-4-5",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "Analyze this tender document for compliance issues"}
    ]
  }'
When to Use Raw API
Direct HTTP Access Scenarios

Languages without official SDK support, serverless environments with strict dependency constraints, learning/understanding how Claude's API works internally, or custom HTTP clients and proxy scenarios.

Claude SDK

The Developer-Friendly Wrapper

#

What is it?

Anthropic provides official SDKs for Python, TypeScript, Java, Go, C#, Ruby, and PHP. These SDKs wrap the API, providing type safety, automatic retries, streaming support, and error handling.

Important Note
C# SDK Version History

As of version 10+, the Anthropic NuGet package is the official SDK. Earlier versions (3.x and below) were the community-built tryAGI SDK, which has moved to tryAGI.Anthropic.

Installation
dotnet add package Anthropic
Basic Usage (C#)
using Anthropic;
using Anthropic.Models.Messages;

AnthropicClient client = new();

MessageCreateParams parameters = new()
{
    MaxTokens = 1024,
    Messages =
    [
        new()
        {
            Role = Role.User,
            Content = "Analyze this procurement document for compliance",
        },
    ],
    Model = Model.ClaudeSonnet4_5_20250929,
};

var message = await client.Messages.Create(parameters);
Console.WriteLine(message);
Streaming Support
await foreach (var chunk in client.Messages.CreateStreaming(parameters))
{
    Console.Write(chunk);
}

06

API Key vs Auth Token

Understanding Authentication

This is one of the most confusing aspects of Claude authentication. Let's clarify the differences.

üîë

API Key (ANTHROPIC_API_KEY)

A static secret string generated in the Anthropic Console. Format: sk-ant-api03-... Sent via the x-api-key HTTP header.

  • Long-lived: Doesn't expire unless revoked
  • Pay-as-you-go: Billed based on token usage
  • Programmatic: Designed for server-to-server communication
  • Workspace-scoped: Can be tied to specific workspaces
üé´

Auth Token (ANTHROPIC_AUTH_TOKEN)

An OAuth 2.0 bearer token obtained through authentication flow. Used with the Authorization: Bearer header.

  • Time-limited: Typically expires after 1 hour
  • Subscription-based: Uses your Claude Pro/Max plan
  • User-scoped: Tied to your personal account
  • Interactive: Designed for tools like Claude Code
API Key

When to Use

Backend services and APIs, automated pipelines, production applications, and team/enterprise environments where billing is centralized.

Auth Token

When to Use

Claude Code CLI sessions, personal development with Claude Pro/Max subscription, and interactive tools that can handle OAuth flows.

Common Pitfall
Auth Conflict Warning

If you have both authentication methods configured, you'll see: "Auth conflict: Both a token (ANTHROPIC_AUTH_TOKEN) and an API key (ANTHROPIC_API_KEY) are set. This may lead to unexpected behavior."

Resolution: For Claude Code with subscription, unset ANTHROPIC_API_KEY. For programmatic access, unset ANTHROPIC_AUTH_TOKEN.
