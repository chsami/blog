Claude Code · March 2026

When Your AI Stops Showing Its WorkThe file-path fight in Claude Code is not about file paths.

In February 2026, Anthropic shipped a one-line UI change to Claude Code that collapsed file-path output into an uninformative summary. 398 points on Hacker News, 241 comments, a GitHub thread, and a direct public response from the head of Claude Code later, the debate has not fully resolved. What looks like a UX disagreement is actually the first serious public argument about who defines trust between a developer and an AI agent.

01
The change
Version 2.1.20

Claude Code version 2.1.20 shipped on February 16th. Before it, when Claude read a file, your terminal showed you the file path. After it, you got a summary line. No paths. Just a count.

Before — v2.1.19
▶ Read src/auth/token.ts
▶ Read config/database.yml
▶ Read tests/integration/payment.test.js
▶ Edit src/auth/token.ts
After — v2.1.20
▶ Read 3 files (ctrl+o to expand)
▶ Edit 1 file (ctrl+o to expand)

That is it. Which files? The summary does not say. A GitHub issue labeled "Critical" appeared the same day, drawing dozens of comments, all variants of the same objection: the path is the signal. Without it, the line carries no information. As one person put it directly: "If you are going to display something like 'Searched for 13 patterns, read 2 files,' there is nothing I can do with that information. You might as well not display it at all."

Boris Cherny, the creator and head of Claude Code at Anthropic, entered both the GitHub thread and Hacker News to explain the decision. His reasoning was honest, carefully considered, and built on premises that do not match how most of his paying users work.

02
The reasoning
Why Anthropic made this call
Boris Cherny — Head of Claude Code, Anthropic — HN #46978710
Designing for the trajectory, not the moment

"When we released Claude Code, Sonnet 3.5 was able to run unattended for less than 30 seconds at a time before going off the rails. Now, Opus 4.6 1-shots much of my code, often running for minutes, hours, and days at a time. The amount of output this generates can quickly become overwhelming in a terminal. Users give the model a prompt, and don't want to drown in a sea of log output in order to pick out what matters: specific tool calls, file edits, and so on. We dogfooded it internally for over a month. We cannot build the right things without community feedback. In this case, we missed the mark for a subset of our users."

The logic is coherent. As Claude has gotten more capable, a single session generates far more tool calls than it did a year ago. Terminals have fixed font sizes and inconsistent color support. In a session running for hours, the volume of output can obscure the things that actually need attention: diffs, bash results, the state of work at completion. Anthropic dogfooded the change internally for over a month. Most of their engineers preferred the quieter output.

The change was not arbitrary. It was a considered product decision built on real usage data, from people who use Claude Code the way Anthropic imagines it being used: as a background worker you hand a task and check on when it is done. The problem is that this is not how most of the community uses it yet.

03
Two use cases, one product
Where the trust gap lives

The people who preferred the quieter output and the people who erupted on GitHub are not in the same situation. They are using the same product with completely different assumptions about the relationship between developer and agent.

Anthropic's model Community's model
Session length Minutes, hours, occasionally days. The agent runs autonomously and you review the output at the end Short to medium sessions with active supervision. You watch and intervene when something looks wrong
Role of file paths Noise during execution. What matters is the final diff, not the intermediate steps that produced it Course-correction signal. Seeing which files Claude reads in the first ten seconds tells you whether the session is on track
Trust level Established. The agent has earned enough reliability that you do not need to supervise every tool call In progress. Trust is built by watching the agent work correctly across many sessions, not granted upfront
Codebase profile Greenfield or well-bounded scopes. The model's context matches what is in the repo Large, established codebases. Multiple domains share terminology; domain confusion is a real and common failure mode
Error cost Low per-session cost of a wrong turn, caught at review. Batch output is reviewed before anything is merged High per-session token cost of an undetected wrong turn. Catching it at review means discarding work that consumed real money

"Files containing the word 'payment' or 'onboarding' might be for entirely different DDD domains than the one relevant to the problem. The very first thing Claude does when planning, choosing which files to read, is a key point for manual intervention. Once I know Claude is looking in the right place, I'm on to the next task."

CTO — Hacker News thread #46978710

"I don't want verbose mode. I want Claude to tell me what it's reading in the first three seconds, so I can switch gears without fear it's going to the wrong part of the codebase. I want Claude to be the babysitter, but I want to make sure the babysitter knows where I left the note before I head out the door."

Developer — Hacker News thread #46978710

File visibility is not curiosity. It is course correction before the course is set. In a large monorepo, knowing which files Claude reads in the first ten seconds of a session is exactly how you decide whether to let it continue or stop it, correct the context, and restart. Without that information, the first opportunity to catch a domain mismatch comes at the end of the session, when the work is done and you can see the diff.

04
The fix that broke two things
The verbose mode saga

Anthropic's response to the backlash was to repurpose verbose mode. In the new configuration, enabling verbose shows file paths inline as they always had been. The original behavior, restored via a flag. Problem solved, officially.

The problem with this was that verbose mode already meant something. It meant: dump everything. Full thinking traces, hook output, full subagent transcripts, all of it, directly into your terminal. So Anthropic began stripping things out of verbose mode, version by version, making it progressively less verbose. File paths: kept. Thinking traces: removed. Hook output: removed. Subagent transcripts: partially removed, with more to come.

Group 1
People who wanted file paths

Now have to enable a setting called "verbose" to get output that is not, by any normal definition, verbose. The flag name is misleading and the change is not discoverable. Users who did not read the GitHub thread or the release notes have no way to know the behavior changed or how to restore it. Several pinned to version 2.1.19 and stopped updating.

Group 2
People who were using verbose mode

Lost the actual verbose output. Thinking traces and hook output, which some workflows depend on, moved behind Ctrl+O. Verbose mode became less verbose on their behalf, without their asking, to make room for the people who wanted file paths. One setting now tries to serve two incompatible needs and satisfies neither fully.

The one-boolean config flag that would have solved the original complaint in an afternoon, something like show_file_paths: true, remained unshipped. Instead, a series of releases stripped things out of verbose mode one by one. As the symmetrybreak.ing blog put it: "At some point you've just reinvented a config toggle with extra steps."

"Anthropic during the Super Bowl: we'd never disrespect our users. Anthropic on GitHub: have you tried verbose mode?"

The line from the symmetrybreak.ing post landed on Hacker News and stayed there. It is barbed but it points at something real. Anthropic ran a Super Bowl ad in early 2026 positioning Claude as a tool built around user trust. Six days later, they were in a GitHub thread telling developers that the UI change they objected to was "a way to simplify things so you can focus on what matters." The disconnect between the brand statement and the product decision was the thing people could not let go.

05
The money and the brand
Two dimensions the press missed
Financial
File paths are a cost-control mechanism

Claude Code Max is $200 a month. Claude Code's billing charges per token. When Claude misunderstands your intent and reads the wrong domain's files, every tool call from that point forward is anchored to the wrong context. The session produces work you will reject. The faster you can catch a wrong turn, the less money you spend. Watching which files Claude reads in the first ten seconds is not just a workflow preference. It is how you decide whether to let a session run or stop it and reset the context before it burns tokens on a dead end. Hiding that information by default makes the cost of a misaligned session higher, not lower.

Reputational
The Constitutional AI company hides what its AI does

Anthropic's public position is that it takes transparency and safety seriously. Constitutional AI, interpretability research, published red lines on autonomous weapons and mass surveillance. The company's entire competitive identity rests on the argument that it is trustworthy in ways others are not. Then their flagship developer product ships a default that hides what it is doing to your codebase. One Hacker News commenter connected this directly: "Defaulting to a posture that allows people to feel they have visibility into where agentic workflows get their context from is, arguably, critical to Claude's brand messaging." The word "feel" is worth sitting with. Actual control and the feeling of control are not the same thing.

06
The actual question
Who defines the developer-agent relationship

Boris Cherny wrote something in the HN thread that, in retrospect, is the most telling part of this whole episode: "When we started Claude Code, it was just a few of us using it. Now, a large number of engineers rely on Claude Code to get their work done every day. We can no longer design for ourselves."

That sentence is honest about the problem and partially blind to one of its dimensions. Anthropic's internal users are Anthropic engineers. People who work at the company building the model. They have a fundamentally different relationship to Claude's reliability than a developer at a startup paying $200 a month. When Anthropic's internal users preferred the quieter output, that preference was shaped by a level of trust earned through proximity: knowing the people who wrote the model, seeing how its failures mode in controlled conditions, having colleagues you can ask directly when something goes wrong.

The developers in the GitHub thread do not have that trust. They are building it. Watching file paths is one of the tools they use to build it.

"The hiding stuff is weird because the whole reason you'd want to see what Claude is doing isn't just curiosity. It's about catching when it goes off the rails before it makes a mess. Like when it starts reading through your entire codebase because it misunderstood what you asked for. Abstractions are great until they hide the thing that's about to break your build."

Developer — Hacker News thread #47033622

Anthropic is making product decisions for a future where trust between developers and AI agents is already established. Their users are living in a present where it is still being negotiated, session by session, file read by file read. Both are right about their respective realities. The tension will not resolve until either Claude gets reliable enough that watching the first ten seconds no longer matters, or Anthropic accepts that "reliable enough" is not a threshold that flips on any particular version number.

Until then, a config flag would help a lot of people. The ask was never complicated.

The bottom line
The relationship between a developer and an AI agent has to be earned. You cannot ship it as a default.

Anthropic built Claude Code for a mode of use that their best users have not yet reached: deep trust, long autonomous sessions, output reviewed at the end rather than supervised throughout. The UI reflects that future. The developers paying $200 a month are living in a different present, where trust is incremental, context errors are expensive, and file paths are not noise but signal. Closing that gap requires either waiting for the model to get better or giving developers the tools to supervise the model while it does. One of those is a product decision. The other is not.
