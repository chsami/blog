Cursor Cloud AgentsYour laptop is no longer the bottleneck.

I used to think the hard part of using AI for coding was writing the right prompt. Then I realized the real constraint was physical: my laptop had to be open, plugged in, and connected. Cursor just removed that constraint entirely. Cloud Agents means the work happens in the cloud — and you manage it from anywhere.

01
The old model was broken
Agent work chained to a single machine
One task at a time

Running an agent locally means you're renting your own CPU and RAM to do the work. You can kick off one long task, but while it runs, your editor slows down, your fan spins up, and you're effectively blocked. Parallelizing across multiple agents meant multiple terminal windows, multiple branches, multiple headaches to coordinate.

Always tethered

Local agents die when your laptop sleeps, your VPN drops, or you close the lid. The work was tied to the uptime of one physical machine. A bug you spotted in a morning meeting had to wait until you were back at your desk, opened the editor, and manually kicked off the fix.

The shift: from monitoring to dispatching

Local agents require you to watch them. Cloud agents let you dispatch and move on. It's the difference between standing over a contractor's shoulder and dropping a task in a shared inbox. The work happens whether or not you're looking.

02
What cloud agents actually do
Three workflows that changed how the Cursor team ships
Workflow 01
Bug fixing

Spot a bug mid-focus? Dispatch a cloud agent from Slack or the editor without breaking your flow. Cursor's team runs multiple models against the same bug simultaneously — the best result wins. For precise, multi-file fixes, consensus across models produces noticeably better output.

Workflow 02
Quick todos

Small tasks that would normally sit on a backlog forever now get dispatched in the morning before you even reach the office. Commuting? Open cursor.com/agents, queue up five tasks, and arrive to finished pull requests.

Workflow 03
Complex features

Iterate on a plan locally in Cursor's plan mode until you're confident in the approach. Then send the plan to a cloud agent and move on to the next problem while it implements. The hard intellectual work stays with you; the mechanical execution moves to the cloud.

03
The plan → cloud handoff
Where local and cloud meet
Plan mode is the interface

Cursor's plan mode lets you iterate with a model to build a detailed implementation plan — which files change, which functions to write, how the pieces connect. Once the plan is solid, you can send it directly to a cloud agent for execution. You stay in creative-mode. The agent handles production-mode.

04
Multi-model consensus
Running several agents on the same problem
Why it works

Hard bugs often require a precise combination of changes across multiple files. No single model gets it right every time. Cursor's team regularly runs two or three models against the same bug in parallel and picks the best result. The cloud makes this trivially cheap — no local resource contention, no waiting for the first run to finish before starting the second.

GPT-5 Codex in the cloud

Cursor has revamped the GPT-5 Codex agent harness specifically for long-running cloud tasks. Local sessions time out, lose context, or hit memory limits on extended runs. The cloud harness is built for persistence — tasks that take 30 minutes or more stay coherent from start to finish.

What this changes about your day
The compounding effect of async delegation
Morning dispatch

Before your first coffee, open cursor.com/agents and queue up the backlog items you've been avoiding. Small migrations, test coverage gaps, config cleanup. By the time you've settled in, several of those tasks are already open as pull requests waiting for your review.

Bug triage without the context switch

A bug surfaces in Slack. Instead of opening the editor, breaking your current focus, and spending 45 minutes on a fix you could review in 5, you dispatch a cloud agent from the Slack thread itself. Come back to it when you're ready. The fix is already there.

The irreversible pattern shift

The teams adopting cloud agents aren't just moving faster — they're working differently. Smaller, more frequent commits. Less context-switching. Faster time from "I noticed this problem" to "this is fixed." Cloud agents don't just speed up existing workflows; they create workflows that weren't previously possible.
