The feature that makes or breaks voice tools

It knows when you stop

This sounds obvious but it is genuinely hard to get right. Deepgram's VAD (voice activity detection) does not cut you off mid-sentence and does not keep the stream open forever after you go quiet. It nails the endpoint.

üîï

Handles background noise well

Keyboard clatter, fan noise, the occasional notification ping ‚Äî none of it triggers false positives. When you are coding, your environment is never completely silent and Deepgram handles that reality gracefully.

Low latency streaming

Using the WebSocket streaming API you get interim transcription results as you speak. By the time you finish a sentence the final transcript is already there. There is no perceivable wait after you stop talking.

Python ‚Äî streaming transcription with endpointing
import asyncio
from deepgram import DeepgramClient, LiveTranscriptionEvents, LiveOptions

async def transcribe():
    dg = DeepgramClient("YOUR_API_KEY")
    conn = await dg.listen.asynclive.v("1")

    async def on_message(self, result, **kwargs):
        sentence = result.channel.alternatives[0].transcript
        if result.is_final and sentence:
            print(f"Final: {sentence}")

    conn.on(LiveTranscriptionEvents.Transcript, on_message)

    options = LiveOptions(
        model="nova-3",
        language="en-US",
        smart_format=True,
        endpointing=300,   # ms of silence to trigger end-of-utterance
    )

    await conn.start(options)
    # feed your microphone stream here
    await conn.finish()

asyncio.run(transcribe())
