/remote-controlStart a Claude Code session locally. Continue it from your phone.

I was three hours deep into a refactor when my dog started doing the "I need to go outside right now" dance. Normally that means losing context. With /remote-control, I typed one command, pulled out my phone, and kept reviewing Claude's output while walking through the park. The session never stopped.

01
The problem it solves
Your terminal is not portable
Chained to the desk

Claude Code sessions can run for a long time. You kick off a migration, a big refactor, or a test suite and then you wait. But you also need to eat lunch, take a walk, or let the dog out. Until now, you had two options: let it run blind and hope for the best, or sit there watching the terminal scroll.

Context is expensive

Stepping away isn't just about missing output. It's about losing your place in the conversation. When Claude asks a clarifying question and you're not there to answer, the whole session stalls. You come back ten minutes later, re-read everything, and try to remember where you left off. That mental reload costs more than the walk itself.

02
How /remote-control works
One command, one URL, any device
Three steps, zero configuration

You type /remote-control in your active Claude Code session. It generates a unique URL and displays a QR code right in your terminal. Scan the QR code with your phone or paste the URL in any browser. That's it. You're now looking at your live session on a second screen. 

$ claude
> /remote-control

Remote control enabled.
Scan this QR code or open the URL below:

  https://claude.ai/code/session/a8f3...

  [QR CODE]

Waiting for connection...
Full session access

The remote view isn't a read-only log viewer. You can read Claude's output, see the full conversation history, and send messages back. When Claude asks "should I proceed with this approach?" you can tap "yes" from the park bench. The session continues as if you never left your desk.

Your terminal stays alive

The local session keeps running normally. Your terminal doesn't lock, the process doesn't pause. If you walk back to your desk, you can continue from the terminal and the phone connection gracefully disconnects. Both ends work simultaneously if you want them to.

The real unlock

This isn't about working from your phone. Nobody wants to write code on a 6-inch screen. It's about never being blocked because you stepped away. Approve a plan, answer a question, check progress — then put the phone back in your pocket.

What you can and can't do remotely
It's a remote control, not a remote desktop
What works great
+Read the full conversation and output history
+Send messages and respond to Claude's questions
+Approve or reject proposed changes
+Monitor progress on long-running tasks
+Review diffs and file changes
What's better at the desk
-Writing long, detailed prompts (phone keyboard)
-Reviewing large diffs across multiple files
-Copy-pasting code snippets into the conversation
-Running your own terminal commands alongside Claude
05
Security and session details
How the connection stays safe
Authenticated and encrypted

The remote URL is tied to your Anthropic account. Opening it requires authentication through claude.ai. The session link contains a unique token that expires when the session ends. Nobody can access your session without being logged into your account.

Session lifecycle

The remote connection lives as long as your Claude Code session does. If you close the terminal, the remote view ends. If your phone disconnects (bad signal, screen lock), the session continues running locally. You can reconnect by scanning the QR code again or reopening the URL.
